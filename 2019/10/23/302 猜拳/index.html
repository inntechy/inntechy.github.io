302 猜拳

发表于 2019-10-23 更新于 2019-11-12 分类于 题解
本文字数： 1.6k 阅读时长 ≈ 8 分钟

题意说明

两个同学用电脑猜拳，1代表剪刀，2代表石头，3代表布。两人按固定的套路出拳，因此他们每次的出拳是可以确定的。

题目提供的输入如下：

  • 第一行
    • 整数N：两人总共猜拳的次数
    • 整数LengthA：A同学套路的长度
    • 整数LengthB：B同学套路的长度
  • 第二行：长度为LengthA，内容为A同学的套路
  • 第三行：长度为LengthB，内容为B同学的套路

题目要求我们输出A、B获胜的次数

思路分析

这道题需要我们模拟两人猜拳的过程（显然，猜拳N次，要用到次数为N的循环）。重点在于，如何获取两人在第i次猜拳中的出拳。

这里先思考一个数学问题，如果今天是星期一，那么请问100天后是星期几？相信大家很快就可以想到思路：100 ÷ 7 = 14 ... 2也就是100天后正好过了14个礼拜多两天，因此100天后是星期三。

在这道编程题中，我们可以借鉴这种思路，从而知道第i次猜拳双方各出什么拳。假设i从0开始，使用数组储存的两个套路A[LengthA]和B[LengthB]表示双方的出拳模式。这样，i = 0时，A同学出拳为A[0]，B同学出拳为B[0]。所以，当i为一个不确定的数字时，A同学出拳就是A[i % LengthA]，即套路中，序号为i % LengthA的那个。B同学同理，为B[i % LengthB]。

通过这个思路，我们就知道上述的次数为N的循环中，每一次循环过程里双方的出拳了。这样，最后的工作只需要统计双方获胜的次数，这里用if判断即可，不再赘述。
